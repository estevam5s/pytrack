PyTrack agora tem tema light/dark e rotas novas (junho/2026). Ver também [[pytrack-platform]].

**Sistema de tema (light/dark)** — funciona em TODO o app (site + dashboard, todas as rotas):
- Tokens semânticos do Tailwind (`background`, `surface`, `surface-2`, `card`, `border`, `input`, `foreground`, `text-primary`, `text-secondary`, `muted`) são `rgb(var(--token) / <alpha-value>)` em `tailwind.config.ts` — canais RGB para suportar opacidade (ex.: `bg-surface/40`). Cores de marca (`primary`, `green`, `blue`, `magenta`, `secondary`, `danger`) são hex fixas (valem nos dois temas).
- As variáveis são definidas em `app/globals.css`: `:root` = valores dark (padrão); `html.light` = valores light. O `.markdown`, scrollbar, `.card-gradient`, `.border-gradient` usam `rgb(var(--...))` para adaptar.
- No-FOUC: script inline em `app/layout.tsx` (`THEME_SCRIPT`) lê `localStorage['pytrack-theme']` e seta classe `light`/`dark` no `<html>` antes do paint; `<html>` tem `suppressHydrationWarning`.
- Toggle: `components/theme-toggle.tsx` (sol/lua) — está no header do dashboard, no navbar do site (desktop + ao lado do hambúrguer no mobile), e os botões Escuro/Claro em `/configuracoes/aparencia` (`ThemeToggle` de `settings-actions.tsx`). Todos alternam classe `light`/`dark` + persistem.
- Regra ao adicionar UI: nunca usar `text-white` sobre superfície (some no light) — usar `text-foreground`. `text-white` só sobre fundo colorido sólido (botão primary, badge). Botões de gradiente usam `text-[#09090A]` (escuro fixo sobre o gradiente neon).

**Responsividade mobile / overflow**: causa comum de "cortar" no mobile = Recharts `ResponsiveContainer` e itens de grid/flex sem `min-w-0` estourando a largura. Correções: regra global em globals.css `.recharts-responsive-container { min-width: 0 !important }`; `min-w-0` nos cards/itens de grid que contêm gráficos (ex.: `app/(dashboard)/inicio`); `overflow-x-hidden` no root do dashboard layout. Ao adicionar gráficos/grids novos no mobile, sempre por `min-w-0` no item. O **menu hambúrguer do site** (navbar.tsx): (1) fundo opaco garantido via `style={{ backgroundColor: "rgb(var(--background))" }}` + `bg-background` + `z-[60]`; (2) o `motion.div` do menu (fixed) DEVE ficar FORA do `<header>` (são irmãos num fragment), porque o header recebe `backdrop-blur-md` quando aberto e `backdrop-filter` cria containing block para `position:fixed` — se o menu ficar dentro, ele colapsa para a altura do header (77px) e swith `overflow-y-auto` some/clipa todas as rotas. Mesma regra vale p/ qualquer fixed dentro de ancestral com transform/filter/backdrop-filter. **Logo** (`public/logo.png`) aparece no topo do hero da home (`hero-section.tsx`) e no `page-hero.tsx` (rotas /recursos, /trilhas, /precos, /sobre).
