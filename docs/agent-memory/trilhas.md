Rota de trilhas personalizadas no dashboard: **/minhas-trilhas** (label "Trilhas" no nav, grupo Estudar; `/trilhas` é do site público, por isso o dashboard usa `/minhas-trilhas`). Detalhe em `/minhas-trilhas/[id]`.

**Dados**: `lib/trilhas.ts` (client-safe, NÃO importa o manifest) — `TRILHAS[]` com {id,title,subtitle,goal,icon,tier,areas,level,accent}. 12 trilhas mapeadas por **área** dos 74 módulos: free=Primeiros Passos (Fundamentos); essencial=Python Developer/Backend/Análise de Dados/Automação; completo=Engenharia de Dados/ML & IA/DevOps/Arquitetura/IoT/Especializações + **Trilha Completa** (areas=["*"], a "mais completa", R$19). Helpers: `moduleTier(area,slug)` (free=parte-1-basico; completo se area ∈ `COMPLETO_AREAS`=[ML, Eng. de Dados, IoT, Arquitetura, Especializações, Segurança]; senão essencial), `canAccess(userTier,req)`, `TIER_RANK`, `moduleInTrilha`.

**UI**: `components/dashboard/trilhas-view.tsx` (client, filtro por plano, cards com tier badge/stats/progresso/lock) recebe `stats` (calculado no server a partir de MODULES+progress) + `userTier` (icons NÃO podem cruzar server→client, então o view importa TRILHAS direto). Detalhe lista módulos com progresso + lock por módulo.

Free só vê o 1º módulo (middleware). `/minhas-trilhas` está em `isFreeDashboardPath` (navegar/escolher é livre; módulos travam).
