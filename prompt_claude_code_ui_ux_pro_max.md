- Detecte inconsistências visuais.

- Detecte problemas de acessibilidade.
- Detecte componentes duplicados ou visualmente desalinhados.

## 2. Plano de redesign

Crie internamente um plano de redesign com:

- Direção visual escolhida.

# DIREÇÃO VISUAL RECOMENDADA

Use uma direção visual premium com:

- Cards com bordas refinadas e sombras suaves.

- Ícones consistentes.

# ANIMAÇÕES E MICROINTERAÇÕES

Adicione animações completas, fluidas e profissionais em:

- Entrada de seções
- Hover de cards
- Hover de botões
- Focus de inputs
- Transições de menus
- Abertura e fechamento de modais
- Dropdowns
- Sidebars
- Toasts
- Skeleton loading
- Mudança de abas
- Estados vazios
- Feedback de ações

Regras:

- Animações devem ser suaves, rápidas e discretas.
- Use easing profissional.
- Respeite `prefers-reduced-motion`.
- Não crie animações que atrapalhem usabilidade.
- Não prejudique performance.
- Evite layout shift.

Duração recomendada:

- Microinterações: 120ms a 220ms
- Entradas de seção: 300ms a 600ms
- Modais/drawers: 180ms a 320ms

---

# RESPONSIVIDADE

Garanta experiência excelente em:

- Mobile pequeno
- Mobile grande
- Tablet
- Notebook
- Desktop
- Telas largas

Melhore:

- Grids responsivos
- Espaçamentos por breakpoint
- Tamanho de fonte por tela
- Navegação mobile
- Cards empilhados
- Tabelas responsivas
- Menus e sidebars
- Formulários em telas pequenas
- Áreas com overflow

Nenhuma rota deve quebrar visualmente em telas menores.

---

# ACESSIBILIDADE

Melhore sem alterar o funcionamento:

- Contraste de texto
- Estados de foco visíveis
- Navegação por teclado
- Labels e aria quando necessário
- Tamanho mínimo de áreas clicáveis
- Semântica visual
- Legibilidade
- Feedback de erro
- Compatibilidade com `prefers-reduced-motion`

Não remova atributos de acessibilidade existentes.

---

# PERFORMANCE

As melhorias visuais devem manter ou melhorar performance.

Evite:

- Bibliotecas pesadas desnecessárias
- Animações custosas
- Re-renderizações desnecessárias
- Imagens sem otimização
- CSS duplicado
- Código visual espalhado sem padrão

Prefira:

- CSS/Tailwind otimizado
- Componentização reutilizável
- Tokens centralizados
- Animações com transform e opacity
- Lazy loading quando já existir padrão no projeto
