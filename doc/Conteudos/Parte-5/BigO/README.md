# Big O, Complexidade e Performance em Python

Esta trilha organiza o conteúdo de Big O por categorias profissionais. O arquivo original `01_conceitos_completo.md` foi mantido como referência monolítica, mas o estudo principal deve seguir os diretórios abaixo.

O objetivo é evoluir da intuição de crescimento até análise avançada de algoritmos, estruturas de dados, recursão, grafos, programação dinâmica, greedy, algoritmos de strings, custo real em Python, profiling, sistemas reais e estudos de caso.

## Sequência Recomendada

1. Entenda crescimento assintótico e notações.
2. Aprenda a calcular Big O em loops, condicionais, espaço e casos.
3. Domine os custos reais das estruturas de dados em Python.
4. Estude recursão e recorrências.
5. Analise algoritmos clássicos.
6. Aplique otimizações em problemas práticos e sistemas reais.
7. Avance para provas, limites inferiores e classes de complexidade.
8. Meça desempenho com profiling e resolva estudos de caso.
9. Revise estruturas e algoritmos avançados para consolidar a cobertura profissional.

---

## Competências Esperadas

Ao concluir esta trilha, você deve saber:

- explicar Big O, Big Omega, Big Theta, little-o e little-omega;
- comparar classes de crescimento;
- calcular complexidade de loops simples, aninhados, triangulares e logarítmicos;
- analisar tempo e espaço;
- diferenciar melhor, médio, pior caso e caso esperado;
- entender custos de `list`, `tuple`, `dict`, `set`, `deque`, `heapq`, `str`, `bytes` e `bytearray` em Python;
- escolher entre pilhas, filas, deques, árvores, grafos, heaps e hash tables;
- resolver recorrências comuns;
- aplicar o Teorema Mestre;
- analisar busca linear, busca binária, BFS, DFS, Dijkstra, A*, ordenação, grafos, strings, programação dinâmica, greedy e backtracking;
- reconhecer armadilhas frequentes;
- otimizar com estruturas melhores, prefix sums, two pointers e sliding window;
- medir código com `timeit`, `cProfile` e benchmarks cuidadosos;
- relacionar Big O com banco, rede, cache, I/O e sistemas reais.
