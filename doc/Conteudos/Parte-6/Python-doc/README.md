# Documentação em Python

Trilha completa e progressiva sobre documentação em projetos Python: Sphinx, MkDocs e Swagger/OpenAPI.

Documentação profissional não é apenas explicar código. Ela reduz custo de manutenção, acelera onboarding, formaliza contratos, registra decisões, melhora suporte, ajuda testes e torna APIs consumíveis com segurança.

## Competências Esperadas

Ao concluir esta trilha, você deve saber:

- organizar documentação para bibliotecas, APIs e aplicações;
- escrever docstrings úteis;
- gerar documentação de API Python com Sphinx;
- publicar documentação com Read the Docs ou GitHub Pages;
- criar portais com MkDocs e Material for MkDocs;
- documentar APIs REST com OpenAPI;
- personalizar Swagger UI/ReDoc;
- validar contratos de API;
- manter documentação versionada junto ao código;
- automatizar build e publicação em CI/CD.

## Tipos de Documentação

- **Tutorial**: guia passo a passo para iniciantes.
- **How-to**: receita para resolver uma tarefa específica.
- **Reference**: descrição precisa de APIs, classes, funções e configurações.
- **Explanation**: contexto, arquitetura, decisões e trade-offs.

Uma documentação madura separa esses tipos em vez de misturar tudo em uma página enorme.
